# YouTube Creator OS Manual

## What this system is
This vault structure is designed to run your channel like a production studio. It organizes:
- ideas
- research
- guests
- scripts
- thumbnails
- descriptions
- repurposing
- analytics
- planning boards
- prompt library

## Core philosophy
1. Never start from zero.
2. Every video or livestream should have a note.
3. Every guest should have a profile.
4. Every prompt should be reusable.
5. Every published piece should generate derivative content.
6. Every performance result should inform the next upload.

## Main notes and what they do

### 1) YouTube Creator OS
The main hub with the channel strategy, workflow map, template map, prompt map, and the entry canvases.
Use this as the first note you open each day.
Color meaning in the entry workflow: blue = process, pink = human handoff, green = success, yellow = escalation, red = rework.

### 2) Content Calendar Dashboard
Use this for weekly planning, task prioritization, active content tracking, and quick status checks.

### 2b) Daily Digest folder
Use this as the single daily surface where agents leave short updates.
Template: `Daily Digest/Template.md`

### 2c) Agents To Do folder
Use this as the shared daily task queue that agents check before working.
Template: `Agents To Do/Template.md`

### 2d) Personal To Do folder
Use this for your private daily task list.
Template: `Personal To Do/Template.md`

### 2e) Personal Notes folder
Use this for your private daily notes and personal journaling.
Template: `Personal Notes/Template.md`

### 3) Creator Operations Log
Use this as the running log of system changes, process improvements, and build history.

### 4) New Video Starter Pack
Use this when you are starting a fresh video and want the full launch sequence in one note.

### 4b) New Video Project Folder Template
Use this when you want a whole project folder scaffold with local notes for idea, research, script, packaging, QA, repurposing, analytics, and archive.

## Daily workflow

### Morning
- Open the dashboard
- Check the Daily Digest
- Check the Agents To Do
- Check news and trending topics
- Pick the next topic with the strongest urgency and fit
- Create or update a content idea note

### Research block
- Gather sources
- Read relevant articles and transcripts
- Identify the core claim and supporting points
- If there is a guest, update the guest note and create questions

### Script / show prep
- Use the script generator prompt
- Fill the project note
- Add title options and hook options
- Add clip timestamps if useful

### Packaging
- Use the description generator
- Use the thumbnail generator
- Use the caricature prompt if the episode needs a visual poster
- Draft pinned comment and title variations

### Publishing
- Check title, thumbnail, description, chapters, and links
- Schedule or publish
- Add to playlist
- Share on socials or community if relevant

### Repurposing
- Cut clips
- Make community posts
- Make thread / X posts
- Make newsletter text if needed
- Save clip ideas back into the project note

### Analytics
- Check 2h signals
- Check 24h signals
- Check 7d signals
- Log what worked and what should be repeated

## How to use the Kanban boards

### YouTube Content Kanban
Use for video production from idea to archive.
Ideal for one-off videos and evergreen analysis pieces.

### YouTube Livestream Kanban
Use for live episodes, live interviews, and breaking news coverage.
Ideal when the work has a run-of-show and live prep.

### Podcast Guest Kanban
Use for guest acquisition, outreach, booking, prep, and follow-up.
Ideal when people are the bottleneck.

### Repurposing Kanban
Use after publishing to manage clip production and distribution.
Ideal when one source should create many outputs.

## Recommended file naming
- Content notes: clear topic title + date if needed
- Guest notes: guest name
- Prompt notes: action-oriented name
- Boards: format-specific and obvious
- Media exports: `YYYY-MM-DD_topic_asset_v01.ext`
- Derivatives: `short01`, `short02`, `thread01`, `carousel01`

## Best practices
- Keep one note as the source of truth per content piece
- Do not duplicate tasks across too many notes
- Use backlinks to connect guest notes, project notes, and prompt notes
- Update the dashboard daily
- Archive aggressively once the piece is done
- Keep prompt migration clean: Markdown first, TXT only as legacy

## Suggested weekly rhythm
- Monday: topic selection
- Tuesday: research and guest prep
- Wednesday: script and packaging
- Thursday: record or go live
- Friday: edits and clips
- Saturday: repurposing and posting
- Sunday: analytics review and next-week planning

## Troubleshooting
- If the workflow feels heavy, simplify by using only three notes: idea, project, and guest.
- If a topic is hot, skip the idea stage and go straight to project + packaging.
- If a guest is uncertain, keep them in the guest board until confirmed.
- If a board becomes cluttered, archive finished cards and keep the board focused on active work only.
- If you are not sure which template to use, start with the project template.

## Practical workflows

### A. New video workflow
1. Capture the idea.
2. Open the YouTube Content Project Template.
3. Add the topic, angle, hook, title ideas, and source links.
4. Use the script prompt.
5. Use the thumbnail and description prompts.
6. Move the card through the Kanban board.
7. Publish, repurpose, and log results.

### B. New livestream workflow
1. Capture the live topic.
2. Open the YouTube Livestream Template.
3. Define the run of show.
4. Prepare any guest note.
5. Use the livestream Kanban board to manage prep.
6. Go live, save the replay, and cut clips.
7. Update repurposing and analytics.

### C. New guest workflow
1. Create or update the guest note.
2. Move the guest card through the guest Kanban board.
3. Run the guest research prompt.
4. Draft questions, opening lines, and thumbnail cues.
5. After recording, log follow-up notes and future episode ideas.

### D. Repurposing workflow
1. Select the source video or live replay.
2. Add clip candidates to the repurposing board.
3. Produce shorts, posts, and newsletter copy.
4. Link every derivative asset back to the main piece.
5. Log what performed best so you can repeat it.

## Quick start
1. Open the dashboard.
2. If this is a brand new piece, duplicate the project folder template first.
3. Create a content idea or project note.
4. Open the relevant Kanban board.
5. Use the prompt notes to generate supporting assets.
6. Publish and repurpose.
7. Review the log and manual when the process changes.
