# Content Naming Convention

## Why this exists
A consistent naming system makes it easier to search, sort, archive, and repurpose assets.

## Recommended rules

### Content notes
Format:
- `YYYY-MM-DD - Topic - Format`
Examples:
- `2026-05-04 - Iran Crisis - Video`
- `2026-05-04 - Geopolitical Roundtable - Livestream`

### Guest notes
Format:
- `Guest Name`
Examples:
- `Stanislav Krapivnik`
- `Li Jingjing`

### Idea notes
Format:
- `Idea - Topic - Short descriptor`
Examples:
- `Idea - Iran escalation - urgent`
- `Idea - Europe energy - interview`

### Prompt notes
Format:
- `Action - Output type`
Examples:
- `YouTube Description Generator`
- `Podcast Guest Research Generator`

### Media assets
Use a versioned format:
- `YYYY-MM-DD_topic_asset_v01.ext`
Examples:
- `2026-05-04_iran-thumbnail_v01.png`
- `2026-05-04_live-runofshow_v02.pdf`
- `2026-05-04_iran-short01_v01.mp4`

### Clips and derivatives
Use a numbered derivative system:
- `short01`, `short02`, `thread01`, `carousel01`

## Folder suggestions
- `Notes/` for project notes and dashboards
- `Templates/` for reusable note skeletons
- `Prompts/` for prompt notes
- `Attachments/` for exported media

## Standard metadata fields
Recommended properties:
- type
- status
- publish_date
- pillar
- guest
- thumbnail_status
- repurposing_status
- platforms

## Quick test
If you can tell what the file is just by reading the name, the naming system is working.
