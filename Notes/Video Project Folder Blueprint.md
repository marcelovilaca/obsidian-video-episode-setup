# Video Project Folder Blueprint

This note suggests a clean folder structure for one piece of video content.
It is designed to be additive and easy to duplicate without changing the rest of the vault.

## Recommended root structure

Use one folder per video or livestream:

`Projects/Video Projects/YYYY/YYYY-MM-DD - Topic Slug/`

Example:

`Projects/Video Projects/2026/2026-05-04 - iran-us-tensions/`

## Suggested contents inside each project folder

- `00 - Project Hub.md`
- `01 - Idea & Brief.md`
- `02 - Research Dossier.md`
- `03 - Script.md`
- `04 - Packaging.md`
- `05 - Launch QA.md`
- `06 - Repurposing.md`
- `07 - Analytics.md`
- `08 - Archive.md`
- `09 - AI Generation Prompt.md`
- `Assets/`
- `Sources/`
- `Exports/`
- `Clips/`
- `Notes/`

## Why this structure works

- The folder mirrors the full content lifecycle.
- It keeps every note for the project in one place.
- It separates global system notes from project-local notes.
- It makes archive and reuse easier later.
- It supports fast work when the topic is urgent.

## What stays global

Do not copy these into every project folder unless you have a special reason:

- `Templates/`
- `Templates/Bases/`
- `Categories/`
- `Prompts/`
- `Notes/YouTube Creator OS.md`
- `Notes/Content Calendar Dashboard.md`
- `Notes/YouTube Creator OS Manual.md`
- `Notes/Creator Operations Log.md`

Those should remain the shared control layer for the vault.

## One-click workflow

1. Duplicate the folder template: `Projects/Video Projects/_TEMPLATE - New Video Project/`
2. Rename the folder to the new topic and date.
3. Open `00 - Project Hub.md`.
4. Fill the idea, research, script, packaging, QA, repurposing, and analytics notes.
5. Keep assets and exports inside the folder.
6. When finished, mark the project archived.

## Daily use

- Start the day by opening the project hub.
- Work from the note that matches the current stage.
- Do not scatter the same project across many unrelated notes.
- Keep the project folder as the single source of truth for that content piece.

## Suggested naming pattern

- Project folder: `YYYY-MM-DD - topic-slug`
- Hub note: `00 - Project Hub.md`
- Research: `02 - Research Dossier.md`
- Script: `03 - Script.md`
- Packaging: `04 - Packaging.md`
- QA: `05 - Launch QA.md`
- Repurposing: `06 - Repurposing.md`
- Analytics: `07 - Analytics.md`
- Archive: `08 - Archive.md`

## Optional upgrade later

If you want even faster creation, the next step would be a small local automation that duplicates the template folder and pre-fills the topic/date in the filenames.
